# H2H-2Much-Log-Translater
Turning complex network logs into clear, human-readable insights using AI

    Members:

      - Harshitha K

Problem Statement
In real-world systems, network logs are complex, unstructured, and difficult to interpret. During system failures or incidents, engineers must manually analyze large volumes of logs, which increases the time required to identify and resolve issues.

Proposed Solution
We are building an AI-powered pipeline that:
- Parses raw network logs (Syslog, SNMP, VPC logs)
- Detects anomalies and unusual patterns
- Classifies logs into severity levels (Critical, Warning, Informational)
- Converts technical log data into simple, human-readable insights
- Suggests possible actions to resolve detected issues

Objective:
To reduce the Time to Clarity by enabling users to quickly understand system issues without manually analyzing complex logs.

Tech Stack
- Python
- FastAPI / Flask
- Streamlit (Frontend UI)
- OpenAI / Ollama (LLM for insights)

 Features 
- Log parsing and structuring
- Severity classification
- Anomaly detection
- Natural language explanation of logs
- Suggested fixes for issues
- Summary dashboard for quick insights
